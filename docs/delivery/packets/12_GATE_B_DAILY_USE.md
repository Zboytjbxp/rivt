# Packet 12 - Gate B Daily Use

## Objective

Make RIVT worth opening during an ordinary workday without manufacturing activity, attention, or paid-only value. The surface should direct a participant to the exact active job, explain the next useful record, and preserve a truthful boundary between free access and Pro capabilities.

## Scope

1. Read the authenticated participant's real active-work project timeline on Home and show whether a daily log exists for the current day.
2. Keep the next action exact: Home actions must retain the accepted-work id when opening Messages, Photos, Daily log, Records, Estimate, or Invoice.
3. Preserve the existing Shop Talk server-owned community, answer, reaction, report, and notification loops. Do not introduce client-only reputation or fake engagement.
4. Make subscription language match live entitlement enforcement: extended time history and CSV export are Pro capabilities; core work records and field tools remain available on the free plan.
5. Extend operating guidance for moderation, matching alerts, device notifications, and active-work evidence before broader engagement expansion.

## Non-Goals

- No artificial feed posts, fake member counts, fake reputation, or activity-based gamification.
- No job-payment collection, escrow, payroll, or claims that RIVT verifies external funds.
- No scheduled email digest, SMS campaign, or background notification without the required preference, provider, queue, and unsubscribe behavior.
- No new paid feature claim unless an API-level entitlement is enforced.

## Acceptance

- A signed-in participant with active work sees one honest daily-work status sourced from that project timeline.
- If no daily log exists today, the action opens the exact job-scoped Daily Log. If it exists, the Home surface says so without implying completion of unrelated work.
- Project access failure does not block Home or leak a private record; the workspace action remains available.
- Plan copy does not represent generally available records/photos as Pro-only.
- Gate B operators have a documented routine for notification delivery, matching jobs, moderation, support escalation, and the first active-work walkthrough.

## Operations Boundary

The first acceptable daily-use field proof is a legitimate two-participant walkthrough:

1. contractor posts and accepts a genuine job relationship;
2. participant receives and opens the exact work notification;
3. participant captures a job photo and confirms it appears in the scoped project record;
4. participant saves a daily log; Home changes from `No log yet today` to `Daily log saved today`;
5. participant creates an invoice/external-payment record if appropriate;
6. both sides complete and review the work relationship.

Do not create fake production jobs or reviews merely to check this list.
